# RAG 2.0 Chatbot

Streamlit-based RAG chatbot with:
- Project-based chat sessions
- Document upload (TXT/PDF/DOCX)
- Embeddings via `sentence-transformers/all-MiniLM-L6-v2`
- Retrieval from Supabase
- Answer generation via Gemini (`gemini-2.5-flash`)

## Tech Stack

- Streamlit (`app.py`)
- Document processing and retrieval (`document_processor.py`)
- Supabase (messages, embeddings, chat history)
- Gemini API for responses

## Environment Variables

Create a `.env` file in project root:

```env
GEMINI_API_KEY=your_gemini_api_key
SUPABASE_URL=your_supabase_project_url
SUPABASE_API_KEY=your_supabase_legacy_anon_or_service_role_key
YOUR_SITE_URL=http://localhost:8501
YOUR_SITE_NAME=RAG_Chatbot
```

Notes:
- Do not commit real secrets.
- Keep placeholders only in `.env.example`.

## Supabase SQL Setup

Run this in Supabase SQL Editor:

```sql
create table if not exists public.messages (
	id bigint generated by default as identity primary key,
	role text not null,
	content text not null,
	project text not null,
	timestamp timestamptz not null default timezone('utc'::text, now()),
	embedding jsonb
);

alter table public.messages enable row level security;

drop policy if exists "Allow all operations" on public.messages;
create policy "Allow all operations"
on public.messages
for all
using (true)
with check (true);

create index if not exists idx_messages_project_timestamp
on public.messages (project, timestamp);
```

## Run Locally

1. Create and activate virtual environment

Windows:
```bash
python -m venv .venv
.venv\Scripts\activate
```

macOS/Linux:
```bash
python -m venv .venv
source .venv/bin/activate
```

2. Install dependencies

```bash
pip install -r requirements.txt
```

3. Start app

```bash
streamlit run app.py
```

## App Flow

1. Create a project in sidebar.
2. Select a project.
3. Upload document (upload option appears only when a project is selected).
4. Ask questions in chat input.
5. Responses and chat history are saved to Supabase.
6. Refresh browser and re-select project to reload prior chat.

## Friendly Error Cases Implemented

- Missing `GEMINI_API_KEY`
- No document context found for project
- Gemini quota exceeded (`429`)
- Invalid Gemini API key/permissions (`401`/`403`)
- Gemini timeout
- Document decode failure
- Document save/retrieval DB failures

Errors are shown clearly in the UI (chat/sidebar) instead of raw stack traces.
